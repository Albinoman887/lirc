lircmd

This daemon behaves like a MouseSystems Mouse. It gets the received buttons from lircd and convert it into mouse moves. To make this possible, lircmd needs a config file. In this file you have to say which button from which remote causes a mouse move or a mouse button click. Read also the daemon note.

Usage of the Daemons

lircd and lircmd are daemons. You should start them in some init script depending on your system. There are some example sripts for different distributions in the contrib directory.

lircd has its own logfile in /var/log/lircd. lircmd will use syslogd to output error messages. It depends on your system configuration where they will show up.

lircd and lircmd will reread their config files if they receive the HUP signal. Additionally lircd will reopen its logfile so that you can remove old logfiles.

irexec

Run program from keystrokes. The config string consists of the command to be run.

NOTE: If you start irexec, it reads your ~/.lircrc and reacts only on prog= entries which points to irexec. If you have included more than one program in your .lircrc, then start all these programs, they react itself only to their according entries in .lircrc.

irpty

lets you create keystrokes out of the IR codes. E.g., type "irpty ~/.lircrc workbone" to control the CD-player program workbone. Of couse you will have to create an appropriate config file for this purpose first. The config string will be passed to the desired application.

irxevent

infra-red xevent sender. Look at the irxevent.README file for more information. Please note that the part concerning the config file is no longer valid. You will have to use the KEY and BUTTON tokens as config strings in the new config file format.
